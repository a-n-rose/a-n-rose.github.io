Projects

Below is a brief description of the various projects I have worked on or am working on, along with a link to a corresponding blog post.

  • NoIze: Open Source Smart Noise Filter
    • I developed an open source smart noise filter in Python for Prototype Fund (round 5). My manager Peggy Sylopp and I present our work on September 2nd, 2019.
  • Workshop: Train and Implement a CNN, LSTM, CNN+LSTM with Speech Data
    • I led a workshop for PyLadies in Berlin on February 12, 2019. I first provided a background for convolutional neural networks and long short-term neural networks, as well as different speech features relevant for machine learning, especially deep learning. The participants experimented with different feature extraction techniques, visualizing what those looked like, as well as training various models and testing those models with their own speech.
  • Replication Study: I built a clinical language classifier with stacked ConvNet and LSTM neural networks in Python
    • I used freely available speech data, specifically clinical speech data, to see how well a ConvNet+LSTM neural network classifier did as compared with only a ConvNet or LSTM alone. I implemented these models in Python using Keras, with challenging modules such as the TimeDistributed module.
  • Babyname Recommender with KMeans Clustering
    • Using US babyname data, spanning from 1880 - 2017, I built a name recommender that allows users to rate (boy, girl, or all) names, for several different searches. Based off of the user's likes and dislikes, recommendations get made and the performance of those recommendations gets evaluated. In this project I explore the influence of feature selection on the recommender's success.
  • Vocab/Idea Collector, Tester, and Organizer
    • Using SQLite3 I have created a vocab/idea collector with the following functionality:
      • user registration/login (passwords *not* hashed/protected)
      • Easy naming and tagging of new lists
      • Easy adding and tagging of new word-meaning (or similar) pairs
      • Review of word-meaning (or similar) pairs of lists
      • Quiz via flashcards, multiple choice, and fill-in-the-blank
      • Navigation between lists and functionality (add new word/ list)
      • Unittesting
    • Functionality that is in the works:
      • Edit lists
      • Quiz based on performance/ last time quizzed
  • English speech to international phonetic alphabet (IPA) model
    • I am using speech and annotation data from Voxforge, as there are multiple databases of various languages publicly available. I am experimenting with loosely aligning annotation and speech data to see if they can be used to build useful models. If so, I will further explore the annotation and speech data of additional languages. Ideas for uses:
    • Speech to text/IPA: Documents what a person says and can also be repeated via synthesis, i.e. with Espeak.
    • Pronunciation guide: Takes in a word from an unknown language, translates it to IPA, and the IPA gets translated into the user's language of choice (ideally one they know).
  • Build an Emotion (of Speech) Classifier
    • This project is inspired by my desire to see how well neural networks train on MFCC data in emotion classification contexts.
  • Flag Child Speech for SLI:
    • After dipping my toes into the waters of deep learning and speech, I had to see how a long short term memory (LSTM) recurrent neural network did with children's clinical (Specific Language Impairment) vs healthy speech. I'm just starting out on this journey and stoked to see where it leads.
  • Language Classifier: ANN vs LSTM
    • I embarked on this project to learn how to build deep neural networks to identify features in speech. This classifier identifies the language the speaker is speaking. The algorithm is trained thus far on English,German, and Russian.
    • A follow-up project will be to identify features of language pathology via a mobile recording device
  • Mimic Master
    • A game that tests your mimicking skills: it plays a sound and the user mimics it, or tries. If the user can collect enough points, they will be deemed 'Mimic Master!'
    • I generated fingerprints of both the target sound and the user's mimic, and I also compared the pitch patterns of each. The more similar these are, the more points the user earns.
